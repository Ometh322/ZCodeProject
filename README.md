# ♠ Poker Lounge — Tournament Manager

Веб-приложение для управления покерным турниром: таймер, блайнды, игроки — всё в реальном времени на экранах зала.

## Структура

```
poker-club/
├── shared/           # Общие типы, события Socket.IO, пресеты блайндов
├── server/           # Node.js + Express + Socket.IO + Prisma/SQLite
│   └── prisma/       # Схема БД + миграции
└── client/           # React + TypeScript + Tailwind CSS (Vite)
```

## Быстрый старт

### 1. Зависимости

```bash
npm install
```

### 2. Настройка

```bash
# Скопируйте и отредактируйте .env:
copy .env.example .env
```

Обязательно поменяйте пароль:
```
ADMIN_PASSWORD=your-secret-password
```

### 3. База данных

```bash
# Создать SQLite + применить миграцию
npm run db:migrate

# Заполнить тестовый турнир (опционально)
npm run db:seed
```

### 4. Запуск (разработка)

```bash
npm run dev
```

Одна команда поднимает два процесса:
- **Сервер** (Express + Socket.IO) на `http://localhost:4000`
- **Клиент** (Vite dev server) на `http://localhost:5173`

Vite автоматически проксирует `/api` и `/socket.io` на бэкенд.

### 5. Откройте в браузере

| Страница | URL | Описание |
|----------|-----|----------|
| Экран зала | http://localhost:5173/display | Таймер, блайнды, средний стек — для ТВ/проектора |
| Админка | http://localhost:5173/admin | Управление турниром (требует пароль) |
| Вход | http://localhost:5173/login | Страница авторизации |

Пароль для входа — значение `ADMIN_PASSWORD` из `.env` (по умолчанию `change-me`).

## Продакшен-сборка

```bash
npm run build
npm start
```

Собранный клиент раздаётся тем же Express-сервером на `http://localhost:4000` — всё на одном порту, без отдельного Vite.

## Функционал

### Экран зала (`/display`)
- 🎯 Крупный таймер обратного отсчёта (MM:SS)
- 🃏 Текущие блайнды (SB / BB) и анте
- 📋 Следующий уровень (превью)
- 📊 Средний стек, количество оставшихся игроков
- 🟢 Индикатор подключения к серверу
- ⏸ Пульсирующий баннер «Пауза»

### Админка (`/admin`)
- ▶️ / ⏸ Старт, пауза, продолжить
- ⏭ / ⏮ Переключение уровней (вперёд / назад)
- ⏱ +1 мин / +5 мин к текущему уровню
- ⟲ Сброс турнира к началу
- ✏️ Настройка названия и общего количества фишек
- 📐 Редактор структуры блайндов (добавление, удаление, редактирование уровней)
- 🎛 Пресеты структур: **Regular** (20 мин), **Turbo** (10 мин), **Deep Stack** (30 мин)
- 👥 Таблица игроков: добавление, редактирование стека, отметка вылета

### Пресеты блайндов

| Пресет | Длительность | Описание |
|--------|-------------|----------|
| Regular | 20 мин | Классическая структура с плавным ростом |
| Turbo | 10 мин | Быстрые турниры, агрессивный рост блайндов |
| Deep Stack | 30 мин | Медленный старт, максимум игры навыка |

Все пресеты можно загрузить и затем кастомизировать — каждый уровень редактируется вручную.

## Архитектура

- **Сервер — авторитетный источник времени**: таймер тикает на бэкенде (1 сек интервал). Клиенты никогда не считают локально — все экраны показывают одно и то же.
- **Socket.IO**: `state:full` при подключении, `timer:tick` каждую секунду, `level:changed` / `status:changed` при изменениях. Переподключение самовосстанавливается.
- **SQLite**: один файл `server/dev.db`, никаких внешних СУБД.
- **Монорепо**: общие типы и события в `shared/`, используются и сервером, и клиентом без дублирования.

## Переменные окружения

| Переменная | По умолчанию | Описание |
|-----------|-------------|----------|
| `PORT` | `4000` | Порт сервера |
| `ADMIN_PASSWORD` | `change-me` | Пароль для входа в админку |
| `ADMIN_TOKEN_TTL` | `43200` | TTL токена в секундах (12 ч) |
| `DATABASE_URL` | `file:./dev.db` | Путь к SQLite |
| `CORS_ORIGIN` | `http://localhost:5173` | Допустимые origins для CORS |
